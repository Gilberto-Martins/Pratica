definicao

Design Patterns ou padrões de projetos são soluções generalistas para ploblemas recorrentes durante o desenvolvimento de um software. Não se trata de um framework ou um código proto, mas de uma definição de alto nível como um problema comum pode ser solucionado.

A Pattern Language

Criado em 1978, por Christopher, Sara Ishikawa e Murray Silverstein
possui 253 tipos de problemas/desafios de projetos

Formato de um Pattern

  • Problema

Criação

os padões de criação são aqueles que abstraem e/ou adiam o processo de criação dos objetos. Eles ajuam a tornar um sistema independente de como seus objetos são criados, compostos e representados.

Estrututais

Os padores estruraus se preocupam com a forma como classes e objetos são compostos para formar estruturas maiores

Comportamentais

Os padões de Comportamento se concentra nos algoritimos e atribuições de responsabilidades entre os objetos. Eles nçao descrevem apenas padões de objetos ou de classes, mas tambme os padrões de cominicação entre os objetos.

Factory

todas as funções que retornam um objeto, sem a necessidade de chama-las com o new, são consideradas funções Factory(fábrica).

Singleton

O objetivo desse pattern é criar uma unica instancia de uma função construtora e retorna-la toda vez em que for necessario utiliza-la.

Decorator

Uma função decorator recebe uma outra fubção como parametro e estende o seu comportamento sem modifica-la explicitamente.

Observer

É um pattern muito popular em aplicações javascript. A instancia(subscriber) mantém uma coleção de objetos(observers) e notifica todos eles quando ocorrem mudanças no estado.

Module

É um pattern que possibilita organizarmos melhor codigo, sem a necessidade de expor variaveis globais.
